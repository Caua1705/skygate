# SkyGate Project Instructions

SkyGate is a mobile-first airport navigation web app.

The product helps passengers plan routes inside airports before arriving or while already inside the terminal.

## Current goal

The current focus is NOT a landing page.

The current focus is the first functional user app screen in `index.html`.

This first screen must allow the user to:

- choose journey type;
- choose origin;
- choose destination;
- enter boarding time;
- calculate the route;
- see a compact preview/result card.

## UI must be

- mobile-first
- premium
- clean
- calm
- precise
- trustworthy
- easy to use
- inspired by travel apps, Google Maps and SaaS products

## Use

- `assets/design_system.html` as the main visual reference
- `assets/logo.jpeg` as the SkyGate logo
- deep navy as primary color
- teal/cyan for routes and active states
- soft gold only for small positive highlights
- white/off-white/light gray backgrounds
- rounded cards
- large touch-friendly buttons
- clear hierarchy
- subtle animations

## Avoid

- landing page structure
- big marketing hero
- login
- account creation
- dashboard
- full website sections
- institutional navbar
- cyberpunk style
- heavy dark UI
- excessive neon
- cluttered sections
- too many animations

## Important rule

When creating or updating `index.html`, build the actual user app entry screen, not a marketing hero.

The main element should be the route planning form.
